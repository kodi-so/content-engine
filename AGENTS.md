# Agent Instructions

- This React app uses Tailwind utilities for UI styling. Use Tailwind classes in components for new layout and presentation work.
- Do not add new standalone CSS files or feature-specific CSS blocks unless the user explicitly asks for stylesheet changes.
- Reuse existing shared components and styling primitives before introducing new surface area.
- For UI work, do a brief design pass before implementation: identify the user's task, choose the right product pattern, and avoid surfacing backend concepts as generic panels.
- Prioritize calm, beautiful, task-appropriate product UI. Do not default to card-heavy dashboards for settings, admin, or operational workflows; use quieter sections, dividers, tables, inspectors, or forms when they fit better.
- Treat visual quality as part of done. After implementation, inspect the screen at realistic desktop and mobile sizes and fix awkward hierarchy, cramped controls, excessive boxes, unclear copy, or incoherent spacing before finalizing.
